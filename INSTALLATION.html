Sower LMS

Local Installation Guide for Windows

🏠 Target URL: http://lms.local
Legend: 🌐 Internet Required An active internet connection is needed for this step. 💻 Offline This step can be completed without internet access.

Install XAMPP

🌐 Internet Required

XAMPP is an all-in-one package that provides Apache, MySQL, and PHP 8.2 — everything needed to serve the app locally.

  1. Go to https://www.apachefriends.org/download.html
  2. Download XAMPP for Windows – PHP 8.2.x (pick the latest 8.2.x installer).
  3. Run the installer with default settings. When prompted, select at minimum:
    ✓ Apache    ✓ MySQL    ✓ phpMyAdmin
  4. Install to the default path: C:\xampp
  5. After installation, open the XAMPP Control Panel and click Start for both Apache and MySQL.
💡 Pin the XAMPP Control Panel to your taskbar for easy access during development.
3

Install Composer

🌐 Internet Required

Composer is the PHP package manager used by Laravel to manage all server-side dependencies.

  1. Go to https://getcomposer.org/download/
  2. Download and run Composer-Setup.exe.
  3. When the installer asks for the PHP executable path, enter: 
    C:\xampp\php\php.exe
  4. Complete the installation with default settings.
  5. Open a new Command Prompt and verify: 
    composer --version
    Expected output: Composer version 2.x.x ...
4

Install Node.js and NPM

🌐 Internet Required

Node.js is required to compile the Vue 3 + Vite + Tailwind CSS frontend of this application.

  1. Go to https://nodejs.org/
  2. Download the LTS installer (18.x or higher).
  3. Run the installer. Ensure the option "Add to PATH" is checked.
  4. After installation, open a new Command Prompt and verify both: 
    node --version
npm --version
    Both commands should print version numbers.
5

Install Git

🌐 Internet Required

Git is needed to clone the project repository from source control.

  1. Go to https://git-scm.com/download/win
  2. Download and run the Git installer.
  3. Use the recommended defaults. On the "Adjusting your PATH environment" screen, select "Git from the command line and also from 3rd-party software".
  4. Verify the installation: 
    git --version
6

Get the Project Files

Choose one of the two options below. Both result in the project being available at C:\xampp\htdocs\sower-lms.

📁 Option A — Copy the Project Folder (Recommended if you already have the files)

💻 No Internet Needed

Since this installation guide is stored inside the project folder itself, you likely already have all the project files. Simply copy the entire folder into XAMPP's web root.

  1. Locate the project folder on your computer — it is the folder that contains this guide.
  2. Copy the entire folder:
    • Right-click the folder → Copy (or Ctrl + C)
  3. Navigate to C:\xampp\htdocs\ in File Explorer and paste it:
    • Right-click → Paste (or Ctrl + V)
  4. Rename the folder to sower-lms if it is not already named that.

The final path must be:

C:\xampp\htdocs\sower-lms\
  1. Open Command Prompt and navigate into the project: 
    cd C:\xampp\htdocs\sower-lms
💡 Using this option, Git is not required and Step 5 can be skipped.

🔗 Option B — Clone from Git (If you have a remote repository)

🌐 Internet Required
  1. Open Command Prompt and navigate to XAMPP's htdocs folder: 
    cd C:\xampp\htdocs
  2. Clone the repository: 
    git clone <YOUR_REPOSITORY_URL> sower-lms
    Replace <YOUR_REPOSITORY_URL> with the actual Git remote URL.
  3. Enter the project directory: 
    cd sower-lms
📝 This option requires Git to be installed (see Step 5).
7

Create the MySQL Database

  1. Make sure MySQL is running in the XAMPP Control Panel.
  2. Open a browser and go to: http://localhost/phpmyadmin
  3. Click "New" in the left sidebar.
  4. Enter the database name: sower_lms
  5. Select Collation: utf8mb4_unicode_ci
  6. Click "Create".
8

Configure the Environment File

The .env file holds all environment-specific settings such as the database connection and app URL.

  1. In the project directory, copy the example environment file: 
    copy .env.example .env
  2. Open .env with a text editor and update the following values: 
    APP_NAME="Sower LMS"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://lms.local

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=sower_lms
DB_USERNAME=root
DB_PASSWORD=
  3. Save the file.
📝 XAMPP's default MySQL root user has no password. Leave DB_PASSWORD empty unless you've set one manually via phpMyAdmin.

Install PHP Dependencies

🌐 Internet Required

Run Composer to download all Laravel packages and server-side dependencies into the vendor/ directory.

cd C:\xampp\htdocs\sower-lms
composer install

This may take a few minutes on the first run.

Install JavaScript Dependencies

🌐 Internet Required

npm install
11

Generate Application Key

Laravel requires a unique 32-character key used to encrypt session data and other sensitive values. Generate it with:

php artisan key:generate

This command automatically writes the key to the APP_KEY= field in your .env file.

12

Run Migrations and Seeders

Create all database tables and populate them with initial data.

Run Migrations

php artisan migrate

This creates the following tables:

  • users — Application user accounts
  • borrowers — Loan borrower records
  • categories — Loan categories
  • loan_plans — Loan plan definitions
  • loans — Loan records
  • payment_schedules — Scheduled payment entries
  • payments — Payment transactions
  • loan_payments — Loan payment linkages
  • penalties — Penalty records
  • penalty_payments — Penalty payment transactions
  • roles, permissions, and related Spatie tables

Run Seeders

php artisan db:seed

This creates the default user accounts, permissions, and loan categories required to use the application immediately after setup.

13

Build Frontend Assets

Compile the Vue 3 + Tailwind CSS + FontAwesome front-end into production-ready static files:

npm run build

After completion, the compiled assets will be placed in the public/build/ directory and served by Apache.

Configure Apache Virtual Host

A virtual host entry tells Apache to serve the application at http://lms.local.

14.1 — Enable Virtual Hosts in Apache

  1. Open the file: C:\xampp\apache\conf\httpd.conf
  2. Find the following line and remove the # to uncomment it: 
    # Include conf/extra/httpd-vhosts.conf
    It should look like: 
    Include conf/extra/httpd-vhosts.conf
  3. Save the file.

14.2 — Add the Virtual Host Entry

  1. Open the file: C:\xampp\apache\conf\extra\httpd-vhosts.conf
  2. Scroll to the bottom and add the following block: 
    <VirtualHost *:80>
    ServerName lms.local
    DocumentRoot "C:/xampp/htdocs/sower-lms/public"
    <Directory "C:/xampp/htdocs/sower-lms/public">
        Options Indexes FollowSymLinks MultiViews
        AllowOverride All
        Require all granted
    </Directory>
    ErrorLog "C:/xampp/logs/lms-error.log"
    CustomLog "C:/xampp/logs/lms-access.log" combined
</VirtualHost>
  3. Save the file.

14.3 — Enable mod_rewrite

Laravel requires Apache's URL rewriting module.

  1. Open C:\xampp\apache\conf\httpd.conf.
  2. Find the following line and ensure it is uncommented: 
    LoadModule rewrite_module modules/mod_rewrite.so
  3. Save and close the file.

14.4 — Restart Apache

In the XAMPP Control Panel, click Stop then Start for Apache to apply all changes.

15

Edit the Windows Hosts File

The Windows hosts file maps the custom domain lms.local to your local machine's loopback address.

  1. Press the Windows key, type Notepad, then right-click it and select "Run as administrator".
  2. In Notepad, open: 
    C:\Windows\System32\drivers\etc\hosts
  3. Scroll to the bottom and add: 
    127.0.0.1   lms.local
  4. Save the file. (Administrator privileges are required to save.)
⚠️ If you cannot save the hosts file, make sure Notepad was opened as Administrator. You can also try disabling antivirus software temporarily if it blocks the save.
16

Enable PHP Extensions

Some PHP extensions required by Laravel may be disabled by default in XAMPP's configuration.

  1. Open the file: C:\xampp\php\php.ini
  2. Search for and uncomment (remove the leading ;) the following lines: 
    extension=pdo_mysql
extension=mbstring
extension=openssl
extension=tokenizer
extension=xml
extension=ctype
extension=json
extension=curl
extension=fileinfo
extension=zip
  3. Save php.ini.
  4. Restart Apache in the XAMPP Control Panel.
17

Set Directory Permissions

Laravel needs write access to the storage and bootstrap/cache directories to write logs, compiled views, and cache files.

Windows/XAMPP is generally permissive, but if you encounter a permissions error:

  1. In File Explorer, navigate to C:\xampp\htdocs\sower-lms.
  2. Right-click the storage folder → Properties → Security tab.
  3. Ensure the current user has Full control or at least Modify access.
  4. Repeat for the bootstrap\cache folder.

You can also clear all stale caches from Command Prompt:

php artisan cache:clear
php artisan config:clear
php artisan route:clear
php artisan view:clear

Final Verification

Before opening the browser, confirm all steps are complete using the checklist below.

  • XAMPP Apache is running (green indicator in Control Panel)
  • XAMPP MySQL is running (green indicator in Control Panel)
  • .env file exists with correct DB_* settings and APP_URL=http://lms.local
  • vendor/ directory exists (Composer install completed)
  • node_modules/ directory exists (npm install completed)
  • public/build/ directory exists (npm run build completed)
  • APP_KEY is set in .env (artisan key:generate completed)
  • Database sower_lms exists in MySQL / phpMyAdmin
  • Migrations ran without errors
  • Seeders ran without errors
  • Apache virtual host is configured in httpd-vhosts.conf
  • Windows hosts file has 127.0.0.1 lms.local entry
  • mod_rewrite is enabled in httpd.conf

Open the Application

Open a browser and navigate to:

http://lms.local

You should see the Sower LMS login page.

Troubleshooting

"No application encryption key has been specified"

Run php artisan key:generate and make sure APP_KEY is populated in .env.

"SQLSTATE[HY000] [1049] Unknown database 'sower_lms'"

Create the database in phpMyAdmin first, then re-run php artisan migrate.

"Class 'PDO' not found" or MySQL connection errors

Enable extension=pdo_mysql in C:\xampp\php\php.ini and restart Apache.

Apache won't start (port 80 conflict)

Port 80 may be in use by IIS, Skype, or another service. In XAMPP Control Panel → Config → Apache → httpd.conf, change Listen 80 to an alternate port, or stop the conflicting application.

"mix: Vite manifest not found" / blank white page

Run npm run build to generate the compiled assets in public/build/.

500 Internal Server Error

Open storage/logs/laravel.log to read the detailed error message and trace the exact cause.

Virtual host not loading (shows XAMPP default page instead)

Ensure the Include conf/extra/httpd-vhosts.conf line in httpd.conf is uncommented and Apache was fully restarted after the change.

Changes to .env are not reflected

Run php artisan config:clear to clear the configuration cache and force Laravel to re-read the .env file.
